# Brute Force Algorithm

## 1. 브루트포스란?  

**브루트포스(Brute Force)** 는 가능한 모든 경우를 직접 확인하여 정답을 찾는 알고리즘 기법  

문제의 조건을 만족하는 답이 어디에 있는지 알 수 없을 때  
**가능한 모든 경우를 하나씩 시도하면서 정답을 찾는 방식**이다.


**⚡️ 핵심**  
"될 수 있는 모든 경우를 전부 확인한다"  


브루트포스는 가장 직관적인 문제 해결 방식   
알고리즘 문제 해결의 기본적인 접근 방법 중 하나  

<br>

## 2. 브루트포스의 특징

### 1) 모든 경우를 탐색
정답이 될 수 있는 모든 경우를 하나씩 확인

ex.  

- 가능한 모든 수열
- 가능한 모든 조합
- 가능한 모든 위치
- 가능한 모든 경우의 수


### 2) 구현이 비교적 단순
문제 조건을 그대로 코드로 옮기기 때문에 복잡한 알고리즘 없이도 구현 가능  


### 3) 입력 크기가 작을 때 매우 효과적
경우의 수가 많지 않다면 빠르게 해결 가능  
ex. N<=10과 같이 입력 크기가 작은 경우 → 브루트포스로 해결 가능  

### 4) 최적화가 없으면 비효율적일 수 있음  
경우의 수가 많아질수록 탐색 시간이 기하급수적으로 증가  

ex. N=20 → 2^20
이렇게 입력 크기가 커지면 시간 초과 발생 가능성 있음

<br>

## 3. 브루트포스 기본 아이디어

접근 방식
1. 가능한 모든 경우 나열  
2. 각 경우가 문제의 조건을 만족하는지 확인  
3. 조건을 만족하면 정답으로 사용  

> 모든 경우 생성 → 조건 검사 → 정답 갱신

<br>

## 4. 브루트포스의 시간복잡도  
경우의 수만큼 시간이 걸림

| 경우      | 시간복잡도  |
| ------- | ------ |
| 2개의 반복문 | O(N²)  |
| 3개의 반복문 | O(N³)  |
| 부분집합 탐색 | O(2^N) |
| 순열 탐색   | O(N!)  |

<br>

## 5. 브루트포스 사용 가능 기준
| 경우의 수   | 가능 여부     |
| ------- | --------- |
| 10^6 이하 | 대부분 가능    |
| 10^7    | 상황에 따라 가능 |
| 10^8 이상 | 보통 어렵다    |

✚ 다음 조건이 있으면 브루트포스 고려  
- 입력 크기가 작음
- 가능한 경우의 수가 제한적
- 모든 경우를 직접 만들어야 함
- 정답이 하나 또는 여러 개 존재  

<br>

## 6. 브루트포스 문제 유형
1) 모든 경우의 수 탐색 _ 블랙잭, 분해합
2) 조합 / 부분집합 문제 _ 부분집합 합, 특정 조건 만족하는 조합 찾기  
3) 완전탐색 문제 _ 가능한 모든 위치 확인, 가능한 모든 이동 방법 확인  
4) 수열 생성 문제 _ 가능한 모든 수열 생성, 조건을 만족하는 수열 찾기  

<br>

## 7. 브루트포스 장단점  
[장점]
> 구현 직관적  
> 문제 이해 쉬움  
> 작은 입력에서 매우 강력  
> 복잡한 알고리즘 없이 해결 가능  

[단점]  
> 경우의 수 많으면 매우 느림  
> 비효율적인 탐색 발생 가능성  
> 시간 초과 발생 가능성 높음  

## 7. 브루트포스와 백트래킹  
브루트포스는 **모든 경우를 탐색하는 방식**이고,  
백트래킹은 브루트포스에서 **불필요한 탐색을 줄인 방법**  

<br>

## 8. 브루트포스 문제 알아보는 힌트  
- 가능한 모든 경우  
- 모든 경우를 확인  
- 조건을 만족하는 경우 찾기  
- N이 작음  
- 완전탐색
